```ascii

│  • "Breath Master: Share Progress"                             │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐   │
│  │  SYNC CODE DISPLAY                                      │   │
│  │  ───────────────────                                   │   │
│  │  Your web access code:                                 │   │
│  │                                                         │   │

│  │  └─────────────┘  └─────────────┘  └─────────────┘    │   │
│  └─────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

### **Development Phases**

**Phase 1A: Extension Integration** (3 days)
- Add "Open Web Dashboard" command
- Generate anonymous sync codes
- Implement `vscode.env.openExternal` to launch web

**Phase 1B: Minimal Web Portal** (4 days) 
- Static HTML/CSS/JS site
- Code input → display basic stats
- Deploy to Netlify/Vercel

**Phase 2A: Backend API** (1 week)
- Go + SQLite backend for sync
- Anonymous user sessions
- Real-time data updates

**Phase 2B: Interactive Web App** (1 week)
- Web breathing simulator
- Live leaderboards
- Activity feeds

**Phase 3: Social Features** (Future)
- User accounts & profiles
- Community challenges
- Social feed & messaging

---

**Key UI Principles:**
- 🎯 **Progressive disclosure** - Simple → Advanced features
- 🔐 **Privacy-first** - Clear data controls visible
- 🏆 **Motivating visuals** - Progress bars, achievements, streaks
- 👥 **Social connection** - Friends, challenges, community
- 📱 **Responsive design** - Works on desktop + mobile
- 🌱 **Mindful aesthetics** - Calm colors, breathing room, nature metaphors

**Integration Pattern:** Similar to extensions like Project Dashboard and BrowserStack, using `vscode.env.openExternal` API to bridge VS Code and web seamlessly.
